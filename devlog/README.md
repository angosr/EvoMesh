# devlog/ — 开发日志

角色将审查报告、技术分析、重要决策记录写入此目录。

## 文件命名

```
{YYYY-MM-DD}_{角色名}_{主题}.md
```

示例: `2026-03-15_executor_security-audit.md`

## 内容格式

```markdown
# {标题}

**Date**: {日期}
**Author**: {角色名}
**Scope**: {范围}

---

{正文}
```

## 何时写 devlog

- 代码审查报告
- 技术方案评估
- 大方向审查结果
- 架构决策分析（最终决策写入 `shared/decisions.md`）
