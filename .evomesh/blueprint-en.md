# EvoMesh — Strategic Blueprint

> This file is maintained exclusively by the Lead role. Read-only for other roles.

## Vision

A multi-role, self-evolving development framework built on Claude Code. Instead of building a new agent, EvoMesh reuses Claude Code's native capabilities and provides structured role templates, self-evolution protocols, and a Web UI for operations.

## Roadmap

### Phase 1 — CLI + Templates (MVP) ✅
- [x] CLI commands: init / role create|list|delete / start|stop|status|attach / serve
- [x] 3 built-in templates: lead, executor, reviewer (Chinese + English)
- [x] Process management: tmux + node-pty spawn + PID registry
- [x] Multi-account support (CLAUDE_CONFIG_DIR)
- [x] Self-bootstrapping

### Phase 2 — Collaboration + Evolution ✅
- [x] Inbox messaging system
- [x] Self-review protocol (defined in role ROLE.md)
- [x] Lead cross-role review
- [x] Devlog conventions
- [x] Security hardening: P0 command injection fix
- [ ] Runtime config validation (zod) — low priority, configs generated by scaffold
- [ ] Unit test coverage

### Phase 3 — Web UI ← Current
- [x] Terminal bridging (ttyd + WebSocket) — with scrollback and touch support
- [x] Password authentication + login page + session management
- [x] Multi-user management (admin/viewer roles, full CRUD)
- [x] Dashboard layout (resizable 3-panel, tabbed terminals, role status)
- [x] Mobile support (touch events, responsive layout)
- [x] Multi-project management (workspace.yaml)
- [x] Claude Code session auto-resume (--name + session ID)
- [ ] WebSocket auth refresh
- [ ] API security headers / rate limiting

### Phase 4 — Polish (planned)
- [ ] Template import/export
- [ ] Evolution visualization
- [ ] Audit logging
- [ ] Session persistence (currently lost on server restart)

## Architecture Decisions

1. Single-package structure, TypeScript + Express + tmux
2. Compatible with Claude Code's native .claude/ structure
3. All roles work on the same branch and directory
4. tmux process management, foreground + background modes
5. Auth: PBKDF2-SHA512 + in-memory sessions, user data in ~/.evomesh/users.yaml
6. Web UI: Single HTML file SPA, no frontend build dependencies
