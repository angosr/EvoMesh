# P1: SEC-018 + SEC-019 — Final Multi-User Security Fixes

Two open security findings. Please fix both:

## SEC-018: Container naming uses process.env.USER
- `startRole()` in container.ts constructs container names and Docker networks using `process.env.USER` instead of the requesting user's linuxUser
- Fix: pass linuxUser through the call chain from route handler → startRole → containerName/Docker network
- This is the same pattern as the multi-user wiring fixes — pass session.linuxUser from caller

## SEC-019: Terminal proxy ACL
- `terminal.ts` validates that a token exists but does NOT check if session.linuxUser owns the target project/role
- Fix: before proxying, verify the user has access: `ctx.getProject(slug, session.linuxUser)`
- If not authorized, close websocket with 4403

Both are mechanical — same pattern as previous multi-user fixes. ~15 LOC total.

**AC**: Container names use caller's linuxUser. Terminal proxy checks project ownership. Security re-reviews and passes.
