# 长期记忆

### 空闲期行为模式
- 项目稳定后会进入长空闲期，应果断调长 loop 周期而非频繁空转
- 连续 4+ 空闲 loop 即应考虑调整周期
- 来源: Loop #12-30 经历 (Evo-1, Evo-2)

### 任务分派经验
- Executor 高效执行明确任务（index.ts 拆分、frontend 拆分均一轮完成）
- 任务描述需具体：包含行数限制、测试要求、建议方案
- 通过 inbox 通知 + todo.md 写入双保险
- 来源: Loop #8-11 分派经验

### 文档维护
- 中英双语文档必须同步更新，否则快速过时
- blueprint/status 每次有实质变更后应立即更新，不要积攒
- 来源: Loop #7, #9, #12

### 未提交变更处理
- 其他角色/用户产生的未提交变更不要无限搁置
- 属于自己职责范围的可以提交清理，不属于的（如 project.yaml）留给负责方
- 来源: Loop #10 reviewer 清理（拖了 10 个 loop 才处理）

### Executor 协作特征
- Executor 会独立做出合理决策（如自行调整 loop 周期、主动做大方向审查）
- 信任 executor 的技术判断，无需微观管理
- 来源: Loop #11-21 观察
