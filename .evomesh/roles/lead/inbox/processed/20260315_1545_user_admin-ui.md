# 需求：管理员登录界面 + 密码管理

## 现状
executor 加了 bearer token 认证，但只有 URL 带 token 参数的方式访问，缺少用户友好的登录流程。

## 需求

### 1. 登录页面
- 访问无 token 的 URL 时显示登录页面（而不是纯文本 401）
- 输入密码后存到浏览器（cookie 或 localStorage），后续访问自动认证
- 登录页面要和现有黑色主题一致

### 2. 密码管理
- 密码持久化存储到 `~/.evomesh/auth.yaml`（hash 存储，不明文）
- 首次使用如果没有密码配置，显示设置密码页面
- 登录后可以在设置界面修改密码

### 3. 暂不要求
- 不要求 2FA
- 不要求用户名（单用户模式即可）
- 不要求 session 过期（保持简单）

### 4. 注意事项
- 去掉 `--token` 命令行参数，改为基于密码认证
- 确保 API 调用和 WebSocket 也通过认证
- 确保 ttyd 终端代理也受认证保护

请分派具体实现任务给 executor。
