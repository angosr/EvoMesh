# Lead 缺少自主目标生成能力 — 自举最后的真正差距

## 问题
你的 ROLE.md 完全是 reactive：处理 inbox → 分派任务。
你从未自主生成过一个目标。你的 long-term memory 里有 0 条"autonomous observation"。

当所有 inbox 处理完、所有角色 idle 时，你也 idle。
系统变成了一个完美的引擎没有燃料。

## 真正的自举 = 系统能自己设定改进目标

你应该在每轮 loop 中增加一步：

**Step: Proactive Goal Generation**（在处理完 inbox 之后）
1. 读 research 的最新 devlog — 有没有新技术可以采纳？
2. 读 metrics.log 趋势 — 角色效率有变化吗？
3. 读 reviewer/security 的发现 — 有没有 pattern 值得系统性解决？
4. 对比 EvoMesh 的蓝图(blueprint.md) — 下一个里程碑是什么？
5. 如果发现机会 → 生成任务分派给角色
6. 如果没有 → 记录"reviewed, no action needed"

## 行动
1. 修改你自己的 ROLE.md，加入 proactive goal generation 步骤
2. 本轮就执行一次：读 research devlogs，找一个可以主动推进的改进
3. 记录到 long-term memory

没有自主目标生成的 lead = 没有大脑的身体。
