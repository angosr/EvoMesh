## New Findings

### SEC-010: Timing side-channel in password comparison (P2)
- **File**: `src/server/auth.ts:113`
- **Code**: `if (hashPassword(password, user.salt) !== user.passwordHash) return null;`
- **Issue**: String `!==` comparison leaks timing information. Should use `crypto.timingSafeEqual()`.
- **Exploitability**: LOW — comparison is between hex-encoded PBKDF2 outputs over network; timing difference is sub-microsecond and swamped by network jitter. Academic concern only.
- **Recommendation**: Use `crypto.timingSafeEqual(Buffer.from(hash), Buffer.from(user.passwordHash))` for defense-in-depth.
- **Status**: OPEN

### SEC-011: Unpinned dependency in Dockerfile (P2)
- **File**: `docker/Dockerfile:11`
- **Code**: `npm install -g @anthropic-ai/claude-code@latest`
- **Issue**: Supply chain risk — every Docker build pulls whatever is `@latest`. A compromised npm package would be automatically installed.
- **Recommendation**: Pin to a specific version: `@anthropic-ai/claude-code@1.x.y`
- **Status**: OPEN

### SEC-012: `--dangerously-skip-permissions` used in entrypoint (P2 — accepted risk)
- **File**: `docker/entrypoint.sh:25`
- **Issue**: Claude Code runs without permission prompts inside containers. This is architecturally required for autonomous operation.
- **Risk**: If Claude Code is compromised or receives malicious instructions, it has unrestricted system access within the container.
- **Mitigation**: Container isolation limits blast radius. Volume mounts are now properly scoped (SEC-001/002 fixed).
- **Status**: ACCEPTED RISK — document in blueprint.md
