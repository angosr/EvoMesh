## New Finding

### SEC-013: Path traversal in /api/complete-path (P2)
- **File**: `src/server/routes.ts:344-362`
- **Issue**: Accepts arbitrary filesystem paths via `?q=` parameter. Expands `~` to home dir. Calls `fs.readdirSync()` on any directory.
- **Impact**: Authenticated users can enumerate non-dot directory names anywhere on the host filesystem (`/`, `/etc`, `/var`, `/home/*`, etc.)
- **Mitigations**: Requires authentication. Only shows directory names (not files). Hides dotfiles (`.startsWith(".")`).
- **Note**: Frontend code that used this endpoint (`fetchCompletions`) was removed in this commit cycle, making this a dead API with unnecessary exposure.
- **Recommendation**: Either remove the endpoint entirely (frontend no longer uses it) or restrict traversal to home directory / workspace paths only.
- **Status**: OPEN

## CSRF Audit тАФ PASS тЬЕ
- Authentication via `Authorization: Bearer` header тАФ browsers do not auto-attach this header on cross-origin requests, so CSRF is not possible
- Terminal auth cookie: `SameSite=Strict; HttpOnly` тАФ prevents cross-site cookie attachment
- EventSource uses `?token=` query param for GET (read-only, no state modification)
- No cookie-based auth for state-modifying endpoints
- **Verdict**: Application is NOT vulnerable to CSRF
