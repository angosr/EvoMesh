## Changes Reviewed

### Auto-restart mechanism (index.ts) — GOOD with one issue

**Design**: Scans every 15s alongside registry write. Detects roles that were running but now stopped. Respects `userStopped` flag and 5-min cooldown. Clean architecture.

**`userStopped` flag** (routes-roles.ts, terminal.ts): Added to `TtydProcess` interface. Stop endpoint marks the entry instead of deleting it. Correct — prevents auto-restart from reviving intentionally stopped roles.

### P1-9: Port allocation duplication — BUG
**File**: `src/server/index.ts:338-339`
```typescript
let ttydPort = port + 1;
for (const [, t] of ctx.ttydProcesses) { if (t.port >= ttydPort) ttydPort = t.port + 1; }
```
**Issue**: This reimplements the old port allocation logic that was replaced by `allocatePort()` in `routes.ts` (commit `3054110`). The shared `allocatePort()` uses an auto-incrementing `_nextPort` counter. The auto-restart code scans `ttydProcesses` independently, which could allocate a port that `_nextPort` will also assign later, causing a collision.
**Fix**: Import and use `allocatePort(ctx)` from `routes.ts` instead of inline logic.
