## New Findings

### P1 ‚Äî Bug / Logic

#### P1-7: Mission Control crashes if no projects exist
**File**: `src/server/routes.ts:400`
```typescript
if (!requireProjectRole(req, res, ctx.getProjects()[0]?.root || "/", "viewer")) return;
```
**Issue**: When no projects exist, `ctx.getProjects()[0]?.root` is `undefined`, falls back to `"/"`. `requireProjectRole` then calls `hasMinProjectRole` with path `"/"` which will never match any ACL entry. Non-admin users will always get 403.
**Self-attack**: Edge case ‚Äî if no projects exist, there's nothing to show anyway. But the 403 is confusing vs a clean empty response. **Minor.**

#### P1-8: `routes.ts` at 473 lines ‚Äî approaching 500-line limit
**File**: `src/server/routes.ts`
**Issue**: The mission-control endpoint added ~60 lines. At 473 lines, one more feature will break the 500-line rule.
**Suggested**: Extract mission-control into `routes-mission.ts` before it reaches the limit.
